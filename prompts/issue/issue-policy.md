# Issue policy

Only create GitHub Issues after human review of local reports.

Default publish criteria:

- severity: Critical or High
- status: Confirmed or Probable
- repository visibility: not Public, unless explicitly allowed

Every issue must include:

- security fingerprint HTML comment
- severity
- confidence
- status
- affected files
- entry point
- trust boundary
- evidence
- impact
- minimal remediation
- regression test idea

Do not publish full secrets, exploit payloads, or unnecessary sensitive details.
